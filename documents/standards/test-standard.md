# テスト原則

テストは品質保証と円滑な開発フローの両立を目標とする。

## MUST

以下のすべてに準拠すること。

* すべてのテストが通ること。
* 重要なユーザーシナリオがすべてカバーされていること。
* 独立性、並行実行可能性、再現性を維持すること。
* 実装の変更で壊れるような過度に脆いテストを避けること。
* 時間依存・非決定的な待機（sleep等）に依存しないこと。
* テスト間での状態を共有してはならない。

## Should

やむを得ない理由がない限り準拠すること。

* モックの利用は外部依存に限定すること（内部実装のモックは避けること）。
* 得られるメリットにコストが見合わないテストは実行しない。
    * 例） DIVタグの数を数える
